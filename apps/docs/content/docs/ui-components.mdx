# üé® UI Components

Nilovon Starterkit includes a comprehensive UI component library built with **shadcn/ui** and **Radix UI**. All components are fully accessible, customizable, and follow modern design principles.

## üöÄ Quick Start

### **Installation**

The UI components are already included in the `packages/ui` package and can be imported directly:

```typescript
import { Button } from "@repo/ui/components/button";
import {
  Card,
  CardContent,
  CardHeader,
  CardTitle,
} from "@repo/ui/components/card";
```

### **Basic Usage**

```typescript
import { Button } from "@repo/ui/components/button";

export function MyComponent() {
  return (
    <Button variant="default" size="lg">
      Click me
    </Button>
  );
}
```

```typescript
          "border border-input hover:bg-accent hover:text-accent-foreground",
        secondary:
          "bg-secondary text-secondary-foreground hover:bg-secondary/80",
        ghost: "hover:bg-accent hover:text-accent-foreground",
        link: "underline-offset-4 hover:underline text-primary",
      },
      size: {
        default: "h-10 py-2 px-4",
        sm: "h-9 px-3 rounded-md",
        lg: "h-11 px-8 rounded-md",
        icon: "h-10 w-10",
      },
    },
    defaultVariants: {
      variant: "default",
      size: "default",
    },
  }
);
```

## ‚ôø Accessibility

All components are built with accessibility in mind:

- **Keyboard Navigation** - Full keyboard support for all interactive elements
- **Screen Reader Support** - Proper ARIA labels and descriptions
- **Focus Management** - Clear focus indicators and logical tab order
- **WCAG 2.1 AA Compliance** - Meets accessibility standards

### **Accessibility Examples**

```typescript
// Proper labeling
<Label htmlFor="email">Email Address</Label>
<Input id="email" type="email" aria-describedby="email-help" />
<p id="email-help">We'll never share your email with anyone else.</p>

// ARIA attributes
<Button aria-label="Close dialog" aria-describedby="dialog-description">
  <X className="h-4 w-4" />
</Button>
```

## üöÄ Performance

Components are optimized for performance:

- **Tree Shaking** - Only import what you use
- **Lazy Loading** - Components load on demand
- **Memoization** - React.memo for expensive components
- **Bundle Splitting** - Separate bundles for different component groups

## üìö Best Practices

### **Do's**

- ‚úÖ Use semantic HTML elements
- ‚úÖ Provide proper ARIA labels
- ‚úÖ Test with keyboard navigation
- ‚úÖ Use consistent spacing and sizing
- ‚úÖ Follow the design system

### **Don'ts**

- ‚ùå Don't override component styles unnecessarily
- ‚ùå Don't remove accessibility features
- ‚ùå Don't use components for unintended purposes
- ‚ùå Don't ignore responsive design
