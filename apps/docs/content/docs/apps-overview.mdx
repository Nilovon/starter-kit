Nilovon Starterkit includes multiple applications, each designed for specific purposes and optimized for their use cases. This overview explains what each app does and how they work together.

## 🎯 Dashboard App (`apps/app`)

The Dashboard App is a comprehensive Next.js 15 application designed for authenticated users to manage their data and interact with the system.

### **Purpose**

- **User Dashboard** - Central hub for authenticated users
- **Data Management** - CRUD operations for user data
- **Admin Interface** - Administrative functions and user management
- **Real-time Updates** - Live data synchronization

### **Key Features**

- **Authentication Required** - Protected routes and user sessions
- **Responsive Design** - Works on all device sizes
- **Dark/Light Mode** - Theme switching with system preference detection
- **Real-time Updates** - WebSocket integration for live data

### **Technology Stack**

- **Next.js 15** - Latest App Router with React 19
- **React 19** - Concurrent features and improved performance
- **Tailwind CSS 4** - Utility-first CSS framework
- **shadcn/ui** - Beautiful, accessible UI components
- **TypeScript** - Full type safety

### **File Structure**

```
apps/app/
├── src/
│   ├── app/                    # Next.js App Router
│   │   ├── dashboard/          # Dashboard pages
│   │   ├── login/             # Authentication pages
│   │   ├── todos/             # Feature pages
│   │   ├── layout.tsx         # Root layout
│   │   └── page.tsx           # Homepage
│   ├── components/             # React components
│   │   ├── features-section.tsx
│   │   ├── header.tsx
│   │   ├── user-menu.tsx
│   │   └── mode-toggle.tsx
│   ├── lib/                    # Utilities
│   │   ├── auth-client.ts     # Authentication client
│   │   └── utils.ts           # Helper functions
│   └── utils/                  # API utilities
│       └── orpc.ts            # oRPC client setup
├── public/                     # Static assets
└── package.json
```

### **Port Configuration**

- **Development**: http://localhost:3000
- **Production**: Configurable via environment variables

## 🔌 Backend Server (`apps/server`)

The Backend Server is an Elysia-based API server that provides the backend infrastructure for all frontend applications.

### **Purpose**

- **API Endpoints** - RESTful and oRPC endpoints
- **Database Operations** - Data persistence and retrieval
- **Authentication** - User management and sessions
- **Business Logic** - Application-specific operations

### **Key Features**

- **Type-safe APIs** - Full TypeScript integration
- **oRPC Support** - Remote procedure calls with type safety
- **Database Integration** - Drizzle ORM with multiple database support
- **Real-time Capabilities** - WebSocket and event streaming
- **Rate Limiting** - API abuse prevention
- **CORS Support** - Cross-origin request handling

### **Technology Stack**

- **Elysia** - High-performance web framework
- **oRPC** - Type-safe RPC protocol
- **Drizzle ORM** - Type-safe database operations
- **SQLite/PostgreSQL** - Database support
- **TypeScript** - Full type safety

### **File Structure**

```

├── src/
│   ├── db/                     # Database layer
```

## 🚀 How Apps Work Together

### **Data Flow**

```
Web App (Public) → Backend Server → Database
       ↓                ↑
Dashboard App (Auth) → Backend Server → Database
       ↓                ↑
Documentation → Static Content
```

### **Authentication Flow**

```
1. User logs in via Dashboard App
2. Backend Server validates credentials
3. JWT token issued and stored
4. Token used for subsequent API calls
5. Protected routes check token validity
```

### **Development Workflow**

```
1. Code changes in any app
2. Hot reload triggers in development
3. TypeScript checks run automatically
4. Linting and formatting applied
5. Build process optimized with Turborepo
```

### **Shared Resources**

- **UI Components** - Consistent design across apps
- **Type Definitions** - Shared interfaces and types
- **Utility Functions** - Common helper functions
- **Configuration** - Shared build and development configs
